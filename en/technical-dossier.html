Technical dossier

Website engineering

Public documentation of the technical layer that makes elmst.ibaifernandez.com robust internally.

Architecture and hosting

Static HTML/CSS/JS website deployed on Netlify, using serverless functions for runtime config, captcha validation, and contact submit.

  • Hosting: Netlify + production domain `elmst.ibaifernandez.com`.
  • Functions: `runtime-config`, `verify-turnstile`, `submit-contact`.
  • Routing and hardening: `_redirects`, `_headers`, `netlify.toml`.
Quality and CI

Blocking pipeline in GitHub Actions. If one check fails, quality gate fails.

  • Playwright E2E for critical routes and contact form.
  • Desktop/mobile visual regression.
  • Automated accessibility with axe.
  • Lighthouse CI + route-level budgets.
  • Link checker and mixed-content detection.
Security
  • Security headers (CSP, HSTS, X-Frame-Options, etc.).
  • Invisible Turnstile captcha with server-side verification.
  • Honeypot and backend field validation.
  • No legacy PHP backend in production.
Technical SEO and i18n
  • ES/EN versions with page-level `hreflang` and canonical.
  • Sitemap and robots aligned with production domain.
  • Route-level OG/Twitter metadata.
  • Friendly routes with controlled legacy compatibility redirects.
Observability and uptime
  • Sentry for frontend error capture (runtime-config based).
  • External HTTP uptime monitor in UptimeRobot.
  • Alert policy and incident runbook documented.
Continuous operations
  • Manual + weekly automated Quality Gate.
  • Versioned technical report per release.
  • Monthly manual QA CSV checks (desktop/mobile).
  • Backlog and roadmap updated each iteration.
